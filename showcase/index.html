Apache-BigData-SIEM
Lakehouse SIEM Showcase Production Stack
Spark, Hive, Kafka ve Hadoop ile açık kaynaklı, ölçeklenebilir ve gerçek zamanlı güvenlik analitiği mimarisi.

1) Neden "Big Data SIEM"?

Geleneksel SIEM (Security Information and Event Management) çözümleri çoğunlukla veri hacmi bazlı lisanslanır (GB/Gün). Bu durum kurumları, maliyeti düşürmek adına bazı kritik logları silmeye veya görmezden gelmeye zorlar.

Lakehouse mimarisi bu sorunu çözmek için tasarlandı:

  • Sınırsız ve ucuz veri depolama (Hadoop HDFS).
  • Gerçek zamanlı uyarı ve anomali tespiti (Spark Streaming).
  • Geriye dönük yıllar süren hızlı SQL analizleri (Hive).
  • Lisans maliyeti %0, sadece donanım masrafı.

2) Veri Giriş Kapısı: Apache Kafka Nedir?

Apache Kafka, dağıtık, yüksek performanslı bir olay akışı (event streaming) platformudur.

Saniyede milyonlarca log üretildiğinde (DDoS saldırısı, log burst vb.) sistemlerin çökmesini engellemek için bir "tampon" (buffer) görevi görür.

  • Topic: Mantıksal veri kanallarıdır (örn. web-logs, syslogs).
  • Producer: Veriyi üreten sistemler (Flog).
  • Consumer: Veriyi okuyan sistemler (Spark).
  • KRAFT Mimarisi: Artık Zookeeper'a ihtiyaç duymadan kendi kendini yönetebilir.

3) Analiz Motoru: Apache Spark Nedir?

Apache Spark, büyük veri kümelerini bellek (RAM) üzerinde paralel olarak işleyebilen, inanılmaz hızlı bir dağıtık analiz motorudur.

Projelerimizde Spark Structured Streaming kullanarak Kafka'dan akan logları anlık olarak okur, temizler, formatlar (Extract, Transform, Load - ETL) ve depolarız.

  • Micro-batch: Veriyi küçük paketler halinde sürekli işler.
  • Master-Worker: 1 Master yönetir, N adet Worker ağır işi paralel yapar.
  • Fault Tolerance: Bir işçi düşerse iş kaybolmaz, devredilir.

4) Veri Gölü: Apache Hadoop (HDFS) Nedir?

HDFS (Hadoop Distributed File System), devasa boyutlardaki dosyaları standart sunucuların disklerine dağıtarak depolayan bir dosya sistemidir.

Güvenlik logları yıllarca petabaytlarca veri oluşturur. HDFS bu veriyi bloklara böler ve farklı makinelere kopyalayarak saklar.

  • NameNode: Sistemin beyni; hangi dosya hangi makinede, haritayı tutar.
  • DataNode: Gerçek veriyi depolayan işçi sunucular.
  • Replication: Bir sunucu yansa bile veri diğer kopyalardan kurtarılır (Bizim projede X2).

5) Veri Kataloğu: Apache Hive Nedir?

Apache Hive, Hadoop (HDFS) üzerinde depolanan karmakarışık ve ham verilere SQL kullanarak sorgu atmamızı sağlayan veri ambarı katmanıdır.

SIEM analistlerinin Python/Java yazmasına gerek kalmaz; bildikleri "SELECT * FROM logs" komutlarıyla HDFS'teki milyonlarca logu filtreleyebilirler.

  • Hive Metastore: Dosyaların tablo yapısını (şemasını), kolon isimlerini tutar (via PostgreSQL).
  • HiveServer2: Analistlerin uzaktan JDBC/ODBC ile bağlanıp sorgu attıkları servistir.

8) ETL Mantığı (PySpark)

  • Kaynak: Kafka topic aboneliği: web-logs, syslogs, app-logs.
  • İşleme: Regex ile IP, method, path, status; JSON parse ile event/service alanları.
  • Hedef: Hive tablosu siem.logs_parsed ve Parquet formatında kalıcı kayıt.
spark.readStream.format("kafka")
  .option("kafka.bootstrap.servers", "kafka-broker:9092")
  .option("subscribe", "web-logs,syslogs,app-logs")
  .load()
  
  # RegExp & JSON field Extraction
  .withColumn("client_ip", regexp_extract("raw_log", r"^(\\S+)", 1))
  .withColumn("app_event", json_parsed.getField("event"))

  # Yazma Aşaması
  .writeStream
  .foreachBatch(append_to_hive)
  .start()

9) Dağıtık Durum Doğrulama (CLI)

  • HDFS: hdfs dfsadmin -report çıktısında Live datanodes (2) görünmeli.
  • Kafka: Topic listesinde web-logs, syslogs, app-logs görünmeli.
  • Hive: Beeline (JDBC) ile sorgular veri dönmeli.
# Hadoop doğrulama
docker exec -it namenode hdfs dfsadmin -report

# Kafka topic kontrol
docker exec -it kafka-broker kafka-topics.sh --bootstrap-server kafka-broker:9092 --list

# SQL Testi
docker exec -it hive-server2 /opt/hive/bin/beeline -u jdbc:hive2://localhost:10000 \
  -e "SELECT source_topic, COUNT(*) FROM siem.logs_parsed GROUP BY source_topic;"

10) Neden Bu Mimari?

  • Vendor lock-in yok: Tamamen Apache ekosistemi, markaya veya pahalı lisanslara bağımlılık yok.
  • Yatay ölçeklenebilirlik: Daha çok log geldikçe, sadece yeni Kafka Broker veya yeni Hadoop DataNode eklenir.
  • Lakehouse yaklaşımı: Aynı platformda (Data Lake) hem stream analitik (gerçek zamanlı alarmlar) hem de SQL tabanlı forensic sorgu.
  • Sıcak/Soğuk Veri Ayrımı Yok: Loglar devasa HDFS cluster'ı üzerinde olduğu için 3 yıl önceki bir olayı saniyeler içinde SQL ile arayabilirsiniz.

11) Sonraki Adımlar

  • Kafka cluster'ını 3 broker ve topic replication (çoğaltma) ile yüksek erişilebilir hale getirmek.
  • Spark işine watermark ve pencereleme fonksiyonları ekleyerek kural tabanlı saldırı tespiti yazdırmak.
  • Hive tablolarını yıllık/aylık/günlük (partitioning) stratejisiyle fiziksel olarak optimize etmek.
  • Apache Superset gibi bir BI aracı bağlayarak SOC (Security Operations Center) Dashboardları çizdirmek.
